## Teil 2: Zielarchitektur (Industry Standard)

### 2.1 Multi-Agent Patterns (nach LangChain/swarm-rs)

Für deinen Use Case (sequentiell, begrenzte Hardware) empfehle ich das **"Subagents with Sequential Workflow"** Pattern:

```
┌─────────────────────────────────────────────────────────────┐
│                    ORCHESTRATOR (Rust)                       │
│  - Koordiniert Agenten                                      │
│  - Managed State (Markdown)                                 │
│  - Error Handling & Recovery                                │
└──────────────────────┬──────────────────────────────────────┘
                       │
        ┌──────────────┼──────────────┐
        │              │              │
┌───────▼──────┐ ┌─────▼─────┐ ┌──────▼─────┐
│   Analyzer   │ │  Planner  │ │  Executor  │
│   Agent      │ │   Agent   │ │   Agent    │
└───────┬──────┘ └─────┬─────┘ └──────┬─────┘
        │              │              │
        └──────────────┼──────────────┘
                       │
              ┌────────▼────────┐
              │  Obsidian RAG   │
              │  (Vector DB)    │
              └─────────────────┘
```

**Warum sequentiell?**
- Deine Hardware kann nicht gut parallel skalieren
- Einfacheres Debugging (einer nach dem anderen)
- Weniger Race Conditions bei Datei-Operationen

### 2.3 Obsidian RAG Architektur

**Stack:**
- **Embeddings:** BGE-M3 (lokal, 1024 Dimensionen, beste Qualität)
- **Vector DB:** LanceDB oder Chroma (lokal, Datei-basiert)

4. **Generation:** LLM generiert Antwort mit Kontext

```rust
// crates/core/src/state.rs
use serde::{Serialize, Deserialize};
use chrono::{DateTime, Utc};

#[derive(Debug, Serialize, Deserialize)]
pub struct Session {
    pub id: String,
    pub status: SessionStatus,
    pub agent: String,
    pub created_at: DateTime<Utc>,
    pub updated_at: DateTime<Utc>,
    pub parent_session: Option<String>,
    pub context_hash: String,
    pub content: String,  // Markdown-Content
}

#[derive(Debug, Serialize, Deserialize)]
pub enum SessionStatus {
    Todo,
    InProgress,
    Review,
    Done,
    Failed,
}

impl Session {
    pub fn save(&self, vault_path: &Path) -> Result<()> {
        let path = vault_path
            .join("sessions")
            .join(format!("{}.md", self.id));
        
        let content = self.to_markdown()?;
        atomic_write(&path, &content)?;
        
        // Auto-commit nach jeder Änderung
        git_commit(&path, &format!("Update session: {}", self.id))?;
        Ok(())
    }
    
    fn to_markdown(&self) -> Result<String> {
        // YAML Frontmatter + Markdown Content
        Ok(format!(
            "---\n{}---\n\n{}",
            serde_yaml::to_string(&self)?,
            self.content
        ))
    }
}
```

### Phase 3: Obsidian RAG (Wochen 5-6)

**Ziel:** Vault ist durchsuchbar mit semantischer Suche

**Stack:**
- **Vector DB:** LanceDB (Datei-basiert, keine separate DB)
- **Embeddings:** BGE-M3 via Python (beste Qualität)
- **Chunking:** Markdown-aware (512 Zeilen pro Chunk)

**Implementierung:**
```rust
// crates/obsidian/src/rag.rs
pub struct VaultRAG {
    db: lancedb::Connection,
    embedding_model: PythonEmbeddingModel,  // Python-Bridge
}

impl VaultRAG {
    pub async fn index_vault(&self, vault_path: &Path) -> Result<()> {
        let markdown_files = find_markdown_files(vault_path);
        
        for file in markdown_files {
            let chunks = markdown_chunker::chunk(&file)?;
            
            for chunk in chunks {
                let embedding = self.embedding_model.embed(&chunk.content).await?;
                
                self.db.create_table("chunks")
                    .add_row(vec![
                        ("path", chunk.source_path),
                        ("content", chunk.content),
                        ("embedding", embedding),
                        ("metadata", chunk.metadata),
                    ])
                    .execute()?;
            }
        }
        Ok(())
    }
    
    pub async fn search(&self, query: &str, top_k: usize) -> Result<Vec<Chunk>> {
        let query_embedding = self.embedding_model.embed(query).await?;
        
        let results = self.db.search("chunks")
            .vector(query_embedding)
            .limit(top_k)
            .execute()?;
            
        Ok(results)
    }
}
```

```rust
use swarms_rs::SequentialWorkflow;  // Oder eigene Implementierung

pub struct AnalysisWorkflow {
    steps: Vec<Box<dyn Agent>>,
    state: WorkflowState,
}

impl AnalysisWorkflow {
    pub fn new(vault_rag: Arc<VaultRAG>) -> Self {
        Self {
            steps: vec![
                Box::new(StructureAnalysisAgent::new()),
                Box::new(QualityCheckAgent::new()),
                Box::new(SecurityScanAgent::new()),
                Box::new(ReportGenerationAgent::new(vault_rag)),
            ],
            state: WorkflowState::new(),
        }
    }
    
    pub async fn run(&mut self, target: &Path) -> Result<WorkflowResult> {
        let mut context = WorkflowContext::new(target);
        
        for (i, agent) in self.steps.iter().enumerate() {
            println!("Step {}/{}: {}", i+1, self.steps.len(), agent.name());
            
            let output = agent.run(&context).await?;
            context.add_output(agent.name(), output);
            
            // State nach jedem Schritt speichern
            self.state.save_checkpoint(&context)?;
        }
        
        Ok(WorkflowResult::from_context(context))
    }
}
```

## Anhang A: Code-Beispiele

### A1: Atomic Write (Rust)

```rust
use std::fs;
use std::io::Write;
use std::path::Path;
use anyhow::Result;

pub fn atomic_write(path: &Path, content: &str) -> Result<()> {
    // 1. Temp-Datei im selben Verzeichnis erstellen
    let temp_path = path.with_extension("tmp");
    
    // 2. Inhalt schreiben
    let mut temp_file = fs::File::create(&temp_path)?;
    temp_file.write_all(content.as_bytes())?;
    temp_file.sync_all()?;  // Sicherstellen, dass auf Disk geschrieben
    drop(temp_file);
    
    // 3. Atomare Umbenennung
    fs::rename(&temp_path, path)?;
    
    // 4. Verzeichnis sync (Unix-only)
    #[cfg(unix)]
    {
        use std::os::unix::fs::OpenOptionsExt;
        let dir = fs::File::open(path.parent().unwrap())?;
        dir.sync_all()?;
    }
    
    Ok(())
}
```

### A2: Markdown Chunking (Rust)

```rust
pub struct Chunk {
    pub content: String,
    pub source_path: String,
    pub heading: String,
    pub line_start: usize,
    pub line_end: usize,
}

pub fn chunk_markdown(content: &str, max_chars: usize) -> Vec<Chunk> {
    let mut chunks = Vec::new();
    let mut current_chunk = String::new();
    let mut current_heading = String::from("Root");
    let mut line_start = 0;
    let mut line_current = 0;
    
    for line in content.lines() {
        // Neue Überschrift = neuer Chunk
        if line.starts_with("#") {
            if !current_chunk.is_empty() {
                chunks.push(Chunk {
                    content: current_chunk.clone(),
                    source_path: String::new(),  // Wird später gesetzt
                    heading: current_heading.clone(),
                    line_start,
                    line_end: line_current,
                });
                current_chunk.clear();
                line_start = line_current;
            }
            current_heading = line.trim_start_matches('#').trim().to_string();
        }
        
        current_chunk.push_str(line);
        current_chunk.push('\n');
        
        // Chunk zu groß?
        if current_chunk.len() > max_chars {
            chunks.push(Chunk {
                content: current_chunk.clone(),
                source_path: String::new(),
                heading: current_heading.clone(),
                line_start,
                line_end: line_current,
            });
            current_chunk.clear();
            line_start = line_current;
        }
        
        line_current += 1;
    }
    
    // Rest hinzufügen
    if !current_chunk.is_empty() {
        chunks.push(Chunk {
            content: current_chunk,
            source_path: String::new(),
            heading: current_heading,
            line_start,
            line_end: line_current,
        });
    }
    
    chunks
}
```
