## Implementation Notes

### General Principles

1. **Start with Priority 1, item 1**
   - Don't skip ahead
   - Complete each item before moving to next
   - This ensures foundational issues are fixed first

2. **Ask the operator if uncertain**
   - Don't guess on architectural decisions
   - Clarify intent before implementing
   - Document decisions for future reference

3. **Test after each change**
   - Verify the fix works
   - Check for regressions
   - Update this checklist with results

4. **Update documentation**
   - Keep manifest in sync with implementation
   - Document any deviations from original design
   - Add comments explaining "why" not just "what"

### When to Escalate

Escalate to operator if:
- A fix requires architectural decisions
- Multiple valid approaches exist
- The documentation is ambiguous
- A change affects multiple domains
