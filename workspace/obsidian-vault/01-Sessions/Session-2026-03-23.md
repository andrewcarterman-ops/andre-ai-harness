## Security Analysis: Obsidian Second-Brain Setup Prompt

**Date:** 2026-03-23  
**Context:** User shared a comprehensive Obsidian vault setup prompt for Claude Code

### Key Security Concerns Identified:

1. **Global npm Installs (`npm install -g`)**
   - Risk: System-wide installation with potential admin privileges
   - Supply chain attack vector through npm dependencies
   - Difficult to uninstall completely, version conflicts possible
   - Recommendation: Use local installation or npx instead

2. **External Downloads Without Verification (`curl | bash`)**
   - Risk: No integrity checking (SHA256, GPG signatures)
   - Man-in-the-middle attack possible
   - Trust-on-first-use problem
   - Recommendation: Use Git submodules or manual verification

3. **Persistent Background Services (LaunchAgents)**
   - Risk: Run automatically at login, forever
   - Hard to remove if something goes wrong
   - Recommendation: Manual scripts instead of persistent services
