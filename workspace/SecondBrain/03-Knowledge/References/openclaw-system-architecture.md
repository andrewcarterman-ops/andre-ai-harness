## Die 5 Domains

| Domain | Technologie | Verantwortung | Key Files |
|--------|-------------|---------------|-----------|
| **A: Rust ECC** | Rust | Core runtime, memory, tools | `crates/*` (60+ Tests) |
| **B: Node.js Core** | TypeScript | Gateway, sessions, channels | Root Gateway |
| **C: YAML Registry** | YAML | Config, agents, skills, hooks | `registry/*.yaml` |
| **D: PowerShell + Hooks** | PowerShell/Markdown | Automation, events | `hooks/`, `*.ps1` |
| **E: Second Brain** | Markdown | Knowledge, PARA, logs | `memory/`, `second-brain/` |

---

## Interface Boundaries

### A ↔ B: Rust ↔ Node.js
- FFI bindings oder WASM (nicht spezifiziert)
- Tool Registry exposed zu TypeScript
- ECC Runtime managed conversation state

### B ↔ C: Node.js ↔ YAML Registry
- agents.yaml loaded at startup
- skills.yaml scanned before each response
- hooks.yaml triggers handlers

### B ↔ D: Node.js ↔ Hooks/PowerShell
- session:start → hooks/session-start.md
- session:end → hooks/session-end.md
- Cron triggers PowerShell scripts

### B ↔ E: Node.js ↔ Second Brain
- Daily logs: `memory/YYYY-MM-DD.md`
- Curated: MEMORY.md
- Sync: 5-minute cron (questionable)
