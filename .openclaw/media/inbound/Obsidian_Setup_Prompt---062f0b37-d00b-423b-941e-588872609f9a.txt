## STEP 10: Create Topic Notes & Link Files

This is CRITICAL. Without this, conversation exports and imported files appear as orphan nodes in Graph View with no connections to the rest of the vault. Claude Code will create **topic notes** (short notes about recurring people, projects, and tools) and add a **"See Also"** footer to every file with [[WikiLinks]] to relevant topics.

**Why this approach instead of a plugin?** The Automatic Linker plugin modifies text inline — it replaces words inside your content with [[WikiLinks]]. This causes false positives (common words matching filenames) and can corrupt code blocks. The "See Also" footer approach NEVER modifies existing content. It only appends a clean links section at the bottom of each file.

### 10B: Scan Files and Identify Topics

Scan ALL markdown files in `[SCAN_FOLDERS]` (default: Conversations/, Sessions/, Archive/, OpenClaw/, plus any user-added folders from Question 5) using this specific procedure:

```
SCAN PROCEDURE:
1. Read the filenames in [TOPICS_FOLDER]/ — these are your existing topic dictionary
2. For each .md file in [SCAN_FOLDERS]/ (recursively):
   a. Read the file content
   b. Extract from frontmatter: "project" value, "tags" values
   c. Extract from content: capitalized multi-word phrases (e.g., "Claude Code",
      "Olsen Brands Management"), single capitalized words that appear mid-sentence
      (e.g., "using Supabase to..." — "Supabase" is capitalized mid-sentence),
      and text matching patterns like "working on [Name]", "talked to [Name]",
      "using [Name]"
   d. SKIP these common false-positive words (capitalized but not topics):
      However, Also, The, This, Note, Important, Warning, Example, Step,
      Section, Update, See, Here, But, And, Or, If, When, Then, After,
      Before, During, About, Into, From, With, That, What, Which, Where
   e. Record each extracted name and which file it appeared in
3. Count FILE occurrences (not word occurrences — "Supabase" appearing 50 times
   in one file counts as 1)
4. Filter by threshold:
   - Vault has 10+ markdown files: topic must appear in 3+ files
   - Vault has 4-9 markdown files: topic must appear in 2+ files
   - Vault has 1-3 markdown files: create topic notes for any named person,
     project, or tool mentioned (no threshold — too few files to count)
5. Remove any that already have a topic note in [TOPICS_FOLDER]/
6. Categorize remaining into: People, Projects, Tools/Services, Organizations,
   Concepts
7. Present to user for approval (Step 10C)
```

**Topic note creation criteria:**
- **Always create topic notes for:** named people, named projects/products, paid services/tools, organizations
- **Never create topic notes for:** common English words, generic concepts ("debugging", "testing", "the app"), single-use mentions that don't meet the threshold
- **Topic names must be proper nouns or specific names** — "Supabase" yes, "database" no. "ShiftCheck" yes, "checklist" no.
- **Borderline tool/service test:** Would the user want to look up "every conversation where I discussed [X]"? If yes → create a topic note. If [X] is just background infrastructure that appears in every project (like GitHub, React, TypeScript), it's not useful as a hub — exclude it. Good examples: "Stripe" (yes — billing discussions are worth tracking), "MightyCall" (yes — specific service decisions), "React" (no — appears everywhere, not informative), "GitHub" (no — too ubiquitous).

### 10E: Add "See Also" Footers to All Files

For every markdown file in `[SCAN_FOLDERS]` — **scan recursively** (use `**/*.md` or `find -name "*.md"`). The default scan folders are `Conversations/`, `Sessions/`, `Archive/`, and `OpenClaw/`. If the user opted in additional folders in Question 5, include those too. Claude Vault creates a nested `conversations/` subdirectory inside `Conversations/`, so files may be at `Conversations/conversations/*.md`, not just `Conversations/*.md`. Do NOT assume files are only at the top level.

For each file:
1. Read the file content
2. Check which topic names appear in the content (case-insensitive matching)
3. If the file already has a `## See Also` section, update it (don't duplicate)
4. If no topics match, add a comment marker (see footer logic below) so the file isn't re-processed
5. Append to the bottom of the file:

```markdown

---
## See Also
- [[Topic Name 1]]
- [[Topic Name 2]]
- [[Topic Name 3]]
```

**CRITICAL RULES:**
- NEVER modify any existing content in the file — only append to the end
- NEVER add links inside code blocks, URLs, headings, or prose
- Sort links alphabetically within the See Also section
- NEVER remove existing links from a footer — only add missing ones (user may have added manual links)

**FOOTER LOGIC (exact procedure):**

To ADD a new footer (file has no `## See Also`):
1. Check if the file already ends with a `---` horizontal rule. If yes, add `## See Also` after it (don't create a double rule). If no, append `\n\n---\n## See Also\n`.
2. Add matching topic links as `- [[Topic Name]]`, sorted alphabetically.
3. If no topics match, add `<!-- No matching topics found -->` as a placeholder so the file isn't re-processed.

To UPDATE an existing footer (file already has `## See Also`):
1. Find the line containing `## See Also`
2. Read all lines from that point to EOF — these are the existing links
3. Parse existing `[[WikiLinks]]` from those lines
4. Determine what SHOULD be linked (scan file content against [TOPICS_FOLDER]/)
5. Merge: keep all existing links + add any new ones that are missing
6. Replace everything from `## See Also` to EOF with the updated section

### 10G: Set Up Footer Linking LaunchAgent (Ongoing)

New conversation files arrive from Claude Vault as plain markdown with no links. This LaunchAgent runs every 10 minutes and adds "See Also" footers to any new files, keeping the Graph View connected automatically.

**IMPORTANT:** The script below includes the default scan folders (Conversations/, Sessions/, Archive/, OpenClaw/). If the user opted in additional folders in Question 5, add matching glob lines for each (e.g., `"$VAULT_PATH/BotNotes"/**/*.md "$VAULT_PATH/BotNotes"/*.md \`).

Create a shell script at ~/claude-vault/link-new-files.sh:

```zsh
#!/bin/zsh
setopt nullglob  # Prevent literal glob iteration on empty directories

VAULT_PATH="[VAULT_PATH]"
LOCKFILE="$VAULT_PATH/.topic-linker.lock"
TOPICS_DIR="$VAULT_PATH/[TOPICS_FOLDER]"  # Replace [TOPICS_FOLDER] with user's chosen name from Question 4

# Respect lockfile (Claude Code Tier 2 may be running a full re-scan)
if [ -f "$LOCKFILE" ]; then exit 0; fi

# Get list of topic names from filenames in [TOPICS_FOLDER]/
TOPICS=()
for f in "$TOPICS_DIR"/*.md; do
  [ -f "$f" ] || continue
  TOPICS+=("$(basename "$f" .md)")
done

# If no topics exist yet, nothing to link
[ ${#TOPICS[@]} -eq 0 ] && exit 0

# Scan all linked folders for files needing linking
# Default: Conversations/, Sessions/, Archive/, OpenClaw/
# Plus any user-added folders from Question 5 — replace [EXTRA_FOLDERS] with those paths
for f in "$VAULT_PATH/Conversations"/**/*.md "$VAULT_PATH/Conversations"/*.md \
         "$VAULT_PATH/Sessions"/**/*.md "$VAULT_PATH/Sessions"/*.md \
         "$VAULT_PATH/OpenClaw"/**/*.md "$VAULT_PATH/OpenClaw"/*.md \
         "$VAULT_PATH/Archive"/**/*.md "$VAULT_PATH/Archive"/*.md; do
  [ -f "$f" ] || continue

  # If file has a "See Also" with only a comment marker, re-process it
  # (new topics may have been created since the marker was written)
  if grep -q '## See Also' "$f"; then
    if grep -q '<!-- No matching topics found -->' "$f"; then
      # Remove old See Also section so it gets re-processed
      sed -i '' '/## See Also/,$d' "$f"
    else
      # Has real links — skip
      continue
    fi
  fi

  # Scan file content for topic matches (case-insensitive, fixed-string)
  MATCHES=()
  CONTENT=$(cat "$f")
  for topic in "${TOPICS[@]}"; do
    # -F = fixed string (not regex — safe for "C++", "Node.js", ".NET")
    # -i = case insensitive
    echo "$CONTENT" | grep -qFi "$topic" && MATCHES+=("$topic")
  done

  # Add footer
  if [ ${#MATCHES[@]} -gt 0 ]; then
    # Sort matches alphabetically (zsh newline split to preserve multi-word topics)
    SORTED=("${(@f)$(printf '%s\n' "${MATCHES[@]}" | sort)}")
    printf '\n\n---\n## See Also\n' >> "$f"
    for m in "${SORTED[@]}"; do
      printf -- '- [[%s]]\n' "$m" >> "$f"
    done
  else
    # Mark as processed so we don't re-scan every 10 minutes
    printf '\n\n---\n## See Also\n<!-- No matching topics found -->\n' >> "$f"
  fi
done
```

Create a macOS LaunchAgent at ~/Library/LaunchAgents/com.obsidian.topic-linker.plist that:
- Runs the link-new-files.sh script every 10 minutes (StartInterval: 600)
- Starts automatically at login (RunAtLoad: true)
- Logs to ~/claude-vault/topic-linker.log and ~/claude-vault/topic-linker-error.log
Load the LaunchAgent, wait 3 seconds, then verify it's running with `launchctl list | grep topic-linker`.

### 10H: Install Session Safety Hooks

Claude Code supports hooks — shell scripts that fire automatically on session events. Install these hooks to make session logging more reliable and vault context loading automatic.

**Create 3 hook scripts:**

**Script 1: session-safety-net.sh** (fires on session end)
When a session ends — whether by /clear, exit, or logout — this script writes a metadata stub if no full session log was saved. The stub is NOT a replacement for a session log — it's a breadcrumb so you can see a session happened.

```zsh
#!/bin/zsh
# Reads JSON from stdin: { "session_id": "...", "cwd": "...", "source": "...", "duration_seconds": N }
INPUT=$(cat)
CWD=$(echo "$INPUT" | python3 -c "import sys,json; print(json.load(sys.stdin).get('cwd',''))")
DURATION=$(echo "$INPUT" | python3 -c "import sys,json; print(json.load(sys.stdin).get('duration_seconds',0))")

VAULT_PATH="[VAULT_PATH]"
PROJECT=$(basename "$CWD")
TODAY=$(date +%Y-%m-%d)
SESSIONS_DIR="$VAULT_PATH/Sessions"

# Always write a metadata stub — it complements Claude's narrative log
# with machine-verifiable data (duration, files changed)
STUB_FILE="$SESSIONS_DIR/${TODAY}-${PROJECT}-session-stub.md"

# Get files changed during this session (if in a git repo)
FILES_CHANGED=""
if git -C "$CWD" rev-parse --is-inside-work-tree >/dev/null 2>&1; then
  FILES_CHANGED=$(git -C "$CWD" diff --name-only 2>/dev/null | head -20)
fi

DURATION_MIN=$((DURATION / 60))

cat > "$STUB_FILE" << STUB
---
date: $TODAY
project: $PROJECT
tags: [session-stub, $PROJECT]
type: session-stub
duration_minutes: $DURATION_MIN
---

# Session Stub: $PROJECT ($TODAY)

**This is an auto-generated metadata stub, not a full session log.**
If Claude wrote a full session log, it will be a separate file in this folder.

## Metadata
- Duration: ${DURATION_MIN} minutes
- Project directory: $CWD

## Files Changed
$FILES_CHANGED
STUB
```

**Script 2: vault-context-loader.sh** (fires on session start)
Automatically loads recent vault context into Claude's conversation AND injects a reminder to search the vault before going external. This works in tandem with the CLAUDE.md vault-read protocol — the hook provides the data, the protocol provides the behavior rule.

```zsh
#!/bin/zsh
INPUT=$(cat)
CWD=$(echo "$INPUT" | python3 -c "import sys,json; print(json.load(sys.stdin).get('cwd',''))")

VAULT_PATH="[VAULT_PATH]"
PROJECT=$(basename "$CWD")
SESSIONS_DIR="$VAULT_PATH/Sessions"
TOPICS_DIR="$VAULT_PATH/[TOPICS_FOLDER]"  # Replace [TOPICS_FOLDER] with user's chosen name from Question 4

# Get last session log for THIS project (most recent only)
LAST_LOG=$(ls -t "$SESSIONS_DIR"/*.md 2>/dev/null | grep -i "\b${PROJECT}\b" | grep -v "session-stub" | head -1)
LAST_LOG_SUMMARY=""
LAST_LOG_NAME=""
if [ -n "$LAST_LOG" ]; then
  LAST_LOG_NAME=$(basename "$LAST_LOG" .md)
  LAST_LOG_SUMMARY=$(sed -n '/^## Summary/,/^## /p' "$LAST_LOG" | head -5)
fi

# Get recent sessions for THIS project (last 5)
RECENT_PROJECT=$(ls -t "$SESSIONS_DIR"/*.md 2>/dev/null | grep -i "\b${PROJECT}\b" | grep -v "session-stub" | head -5 | while read -r f; do basename "$f" .md; done)

# Get recent sessions ACROSS ALL PROJECTS (last 10, excluding stubs)
# This ensures Claude knows what's in the vault even when CWD doesn't match
RECENT_ALL=$(ls -t "$SESSIONS_DIR"/*.md 2>/dev/null | grep -v "session-stub" | head -10 | while read -r f; do basename "$f" .md; done)

# Get all topic names (use while read to handle apostrophes safely)

VAULT CONTEXT (auto-loaded):

$LAST_LOG_SUMMARY
Recent sessions (this project): $RECENT_PROJECT
Recent sessions (all projects): $RECENT_ALL
Known topics: $TOPIC_NAMES

CONTEXT
```
